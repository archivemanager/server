Application Developers

Remote access to the ArchiveManager api follows a conventional token-based security system. Client developers should log on to receive a secure token which is then included in subsequent calls.

  • 1) The user logs in with a POST request containing his username and password.
    http://servername/service/auth/login?email=&password=
  • 2) The server returns a temporary / permanent authentication token.
  • 3) The user sends the token within each HTTP request via an HTTP header Authorization: Bearer TOKEN.
